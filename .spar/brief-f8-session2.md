## Key decisions binding this session

- **Host-fenced readback (B2)** — cudarc 0.9.15 forces DtoH onto NULL stream (`dtoh_sync_copy`); forked stream and NULL readback stream are genuinely unordered. The fence is load-bearing.
- **A1 (ctx replaces device)** — `CudaImage` becomes `!Send + !Sync`; operations use `ctx.raw_ctx()` for `_Ctx` npp_ctx argument.
- **Session-end-green** — intermediate commits may be red (e.g. Phase 2 renames `device`→`ctx` before Phase 3 updates macro callers). Only the final Phase-6 state must be green.
- **C1 is a two-op smoke test** — not a fence regression guard. cudarc 0.9's context-wide sync makes fence-absent tests mechanically infeasible.

## Reshaped C1 test (Finding B)

The original plan etched a two-readback structure (`Op1 → Readback1 → Op2 → Readback2`).
This session reshapes it to `Op1 → Op2 → single readback`:

```
ctx = StreamContext::new(device)?
img_bgra = CudaImage::<u8>::from_host(ctx.clone(), 4, w, h, bgra_data)?
img_rgb  = CudaImage::<u8>::new(ctx.clone(), 3, w, h)?
img_dst  = CudaImage::<u8>::new(ctx.clone(), 3, w/2, h/2)?

img_bgra.bgra_to_rgb(&mut img_rgb)?          // Op1: 4ch→3ch on forked stream
img_rgb.resize(&mut img_dst, NearestNeighbor)? // Op2: downsample on same stream (no sync)

result: Vec<u8> = Vec::try_from(&img_dst)?     // single readback (fence + NULL-stream copy)
assert_golden(&result, EXPECTED);
```
